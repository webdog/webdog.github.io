Scenario 4: Developer C's source Object just went to PROD but the Developer A and Developer B still are developing it.

In this scenario, we have three developers working off of working branches based on a common master.
Developer C has had a more pressing change to push to PROD. Git makes this manageable through the following steps

  • Developer C communicates the changes to the group
  • Developer A and B continue to work on their branches until they are ready to pull changes down from the updated master
  • Developer A and B both run git fetch and git merge on their local master branches to update Developer C's changes
  • Developer A and B both run git merge master 'working_branch' inside their working branch to incorporate the updated master's changes into their working branch
  • Developer A and B are both up-to-date with Developer C's changes
  • Developer A and B communicate with each other and use the workflow strategy from Scenario 2 to merge their branches
