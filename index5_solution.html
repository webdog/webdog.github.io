Scenario 5: Resolution

:

Resolution Steps


In this scenario, master will not be updated for two weeks, as master is always viewed as PROD. In this case
git can alleviate this pain by the following.

  • Developer A and D communicate that D's code will not ship to master for two weeks
  • Developer A will then create a working branch with Developer D's branch acting as the parent branch
  • Once complete, Developer A will merge their code into Developer D's branch.
  • Once ready for delivery, Developer D will merge back into master for PROD delivery.
