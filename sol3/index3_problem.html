This scenario can be effectively managed in git by Developer A running through the following steps:

  • Developer A communicates with Developer B about Developer B's merge into master
  • Developer A switches out of local working branch into local master
  • Developer A runs git fetch to review the changes upstream from master, and git merge to update the local master.
  • Developer A switches back to local working branch
  • Developer A runs git merge master 'working_branch' to incorporate the changes from master into the working branchh
