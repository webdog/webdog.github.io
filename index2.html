Scenario 2: Developer A starts developing an Object but Developer B already is also developing it.
The scenario as shown in Git.

Resolution 1:
  • Developer A takes the commits from his branched repository off master and commits them upstream to master
  • Developer B switches to their local copy of master
  • Developer B runs git fetch to bring the changes from master upstream into their local repository
  • Developer B runs git merge to merge the upstream master into local master
  • Developer B returns to their local working branch
  • Developer runs git merge master -- This will integrate Dev B's working branch against the updated parent branch master
